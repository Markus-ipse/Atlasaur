# Implementation plans

Per-milestone implementation plans for the retention roadmap (see
`../../RETENTION_ROADMAP.md` for the long-form research and prioritization).

Each file captures the context, design decisions, and verification steps
for one shipped or in-progress milestone — written before implementation,
kept as a record afterwards. Useful when picking the work back up later
(or on a different machine / web session) and for future contributors who
want to know *why* a change was made, not just *what*.
